This activity contains some possible solutions to issues that might arise

(a)
Be careful when pasting the key into the webhook secret field. Be sure that no extra spaces were copied. Push the Update webhook button on github.org after correcting the secret.
(b)
To see updates on the course webpage after pushing changes through git, push the browser’s refresh button and then click the Update activity button on the course page. Deleting your browser’s cache might also resolve certain issues.
(c)
Use \includegraphics{file.png} rather than \includegraphics{file}. LATEX will gracefully handle either case whereas Ximera currently requires explicit file extensions.
(d)
The optional scaling argument of \includegraphics currently does not work on the Ximera side.
(e)
The preamble of every activity file needs to include a \title command.
(f)
The abstract and the \maketitle command of every activity file need to occur after \begin{document}.
(g)
A good way to check for TEX typing mistakes is to run LATEX on all the TEX files. Indeed, this is good practice irrespective of mistakes and additionally produces handouts that can be given to students.
(h)
If your TEX files compile correctly, you should confirm that they have been successfully uploaded to github.org and that github.org has successfully pushed those files in turn to ximera.osu.edu. To do this you can click on the most recent delivery on the Webhooks & Services page in your repository on github.org. In case of any delivery errors, they will be reported here. You can also push the Redeliver button in case of any errors.
